# UC-DOC-003 지원 현황 등록 및 관리

| 항목 | 내용 |
|------|------|
| **액터** | 인증 사용자 (`AuthenticatedUser`) |
| **목표** | 채용공고에 대한 지원 상태를 등록·변경하여 지원 현황을 관리하고, 가상 면접 시작 시 빠르게 선택할 수 있도록 한다 |
| **사전 조건** | 로그인 상태, 채용공고가 1건 이상 등록되어 있음 |
| **사후 조건** | 채용공고에 지원 상태가 저장되어 가상 면접 설정 시 선택지로 제공된다 |

---

## 지원 상태 값

| 값 | 설명 |
|----|------|
| `지원 예정` | 지원할 예정이나 아직 지원하지 않은 상태 (기본값) |
| `관심 저장` | 관심은 있으나 지원 여부 미결정 상태 |
| `지원 완료` | 실제로 지원을 완료한 상태 |

---

## 기본 흐름 — 채용공고 등록 시 지원 상태 설정

1. 사용자가 채용공고 추가 폼에 접근한다 → [UC-DOC-002](./UC-DOC-002-채용공고_등록_및_관리.md)
2. 사용자가 URL, 공고 제목(선택)을 입력한다
3. 사용자가 **지원 상태** 섹션에서 상태를 선택한다 (기본값: `지원 예정`)
4. 사용자가 "채용공고 추가하기"를 클릭한다
5. 시스템이 채용공고와 지원 상태를 함께 저장한다

---

## 대안 흐름 — 채용공고 상세 화면에서 지원 상태 변경

1. 사용자가 채용공고 상세 화면에 접근한다 (15.2 Job Description Detail)
2. 우측 "지원 상태 변경" 카드에서 새 상태를 선택한다
3. 시스템이 즉시 상태를 업데이트한다

---

## 대안 흐름 — 채용공고 수정 화면에서 지원 상태 변경

1. 사용자가 채용공고 수정 화면에 접근한다 (15.3 Job Description Edit)
2. "지원 상태" 섹션에서 상태를 변경한다
3. "저장하기"를 클릭하면 변경된 상태가 저장된다

---

## 대안 흐름 — 지원 현황 삭제

- 채용공고 수정 화면에서 "삭제" 버튼을 클릭하면 `user_job_descriptions` 행이 제거되고 지원 상태도 함께 삭제된다

---

## 비고

- 지원 상태는 별도 페이지 없이 채용공고 등록·상세·수정 화면 안에서 인라인으로 관리된다
- 지원 상태는 `user_job_descriptions` 테이블의 `application_status` 컬럼에 저장된다
- 등록된 지원 현황(채용공고 + 지원 상태)은 [UC-INT-001-면접_설정_및_시작.md](./UC-INT-001-면접_설정_및_시작.md)에서 선택지로 제공된다
- 관련 테이블: `user_job_descriptions`
