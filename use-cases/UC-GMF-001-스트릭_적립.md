# UC-GMF-001 스트릭 적립

| 항목 | 내용 |
|------|------|
| **액터** | 시스템, 일반 사용자, Pro 사용자 |
| **목표** | 가상 면접 세션 완료 시 해당 사용자의 스트릭을 적립한다 |
| **사전 조건** | 가상 면접 세션이 `completed` 상태로 종료됨 |
| **사후 조건** | 오늘 날짜의 스트릭이 기록되고, 연속 참여 일수가 갱신된다 |

---

## 기본 흐름

1. 사용자가 가상 면접 세션을 완료한다
2. 시스템이 해당 사용자의 오늘 날짜 스트릭 적립 여부를 확인한다
3. 오늘 아직 적립되지 않은 경우, 시스템이 스트릭을 처리한다
   - `last_participated_date`를 오늘 날짜로 갱신한다
   - `current_streak`을 1 증가시킨다
   - `current_streak`이 `longest_streak`을 초과하면 `longest_streak`도 갱신한다
   - `streak_participation_logs`에 오늘 날짜를 기록한다 (달력 뷰 데이터 소스)
4. 시스템이 스트릭 보상 달성 여부를 확인한다 → [UC-GMF-004-스트릭_보상_수령.md](./UC-GMF-004-스트릭_보상_수령.md)

---

## 대안 흐름

- **3a.** 오늘 이미 스트릭이 적립된 경우 중복 적립하지 않고 종료한다

---

## 비고

- 스트릭 적립은 면접 세션 완료(`completed`) 시에만 발생하며, 중단(`abandoned`) 세션은 적립되지 않는다
- 연습 모드와 실전 모드 모두 스트릭에 반영된다 (모드 구분 없이 동일하게 적립)
- 하루에 여러 번 면접을 완료해도 스트릭은 1회만 적립된다
- 관련 테이블: `streaks`, `streak_participation_logs`
