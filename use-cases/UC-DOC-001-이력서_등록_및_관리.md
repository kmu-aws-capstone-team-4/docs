# UC-DOC-001 이력서 등록 및 관리

| 항목 | 내용 |
|------|------|
| **액터** | 인증 사용자 (`AuthenticatedUser`) |
| **목표** | 면접 질문 생성에 활용할 이력서를 등록하고 관리한다 |
| **사전 조건** | 로그인 상태 |
| **사후 조건** | 등록된 이력서가 벡터 DB에 저장되어 면접 질문 생성 시 맥락 자료로 활용 가능한 상태가 된다 |

---

## 기본 흐름

1. 사용자가 이력서 관리 페이지에 접근한다
2. 사용자가 이력서 입력 방식을 선택한다

   | 방식 | 설명 |
   |------|------|
   | **파일 업로드** | PDF, DOCX 파일을 업로드한다 |
   | **자유 텍스트 입력** | 경력, 기술 스택, 학력 등을 자유롭게 텍스트로 입력한다. 한 줄도 허용 |
   | **건너뛰기** | 이력서 없이 직군·직업 정보만으로 면접을 시작한다 (이탈 방지) |

3. 파일 업로드 또는 자유 텍스트 입력을 선택한 경우:
   - 이력서 제목(식별용)을 입력한다
   - 시스템이 입력 내용을 `content_text`로 저장한다 (즉시 응답 반환)
4. 시스템이 비동기로 후처리를 수행한다
   - 파일인 경우 텍스트를 파싱·추출한다
   - LLM으로 `parsed_data`(기술 스택, 경력 요약 등)를 추출하여 저장한다
   - 텍스트를 청킹하여 벡터 DB(`user_document_chunks`)에 임베딩 저장한다
   - 사용자의 경향 분석 요약(`user_trend_summaries`)을 갱신한다
5. 이력서가 목록에 추가되고 기본적으로 활성화 상태로 설정된다
6. 후처리 완료 후 UI가 구조화된 뷰(`parsed_data` 기반)로 전환된다

---

## 대안 흐름

- **3a.** 지원하지 않는 파일 형식인 경우 오류 메시지를 표시하고 자유 텍스트 입력을 유도한다
- **3b.** 파일 파싱 실패 시 자유 텍스트 입력을 유도한다
- **4a.** 후처리 완료 전까지 UI에 "분석 중..." 상태를 표시하고, 원문 텍스트 뷰로 폴백한다
- **5a.** 사용자가 기존 이력서를 비활성화하면 면접 질문 생성에서 제외된다
- **5b.** 사용자가 이력서를 삭제하면 벡터 DB에서도 해당 청크가 제거된다

---

## 비고

- 이력서는 필수 사항이다.
- 여러 버전의 이력서를 등록할 수 있으며, 각각 활성화/비활성화로 관리한다
- `resume_type`: `file` | `text` 로 입력 방식을 구분하여 저장한다 (db-schema.md `resumes.resume_type` 참조)
- `parsed_data` (JSONB): LLM이 추출한 구조화 데이터. UI 표시용으로 활용
  ```json
  {
    "skills": ["Python", "Django", "PostgreSQL"],
    "total_experience_years": 3,
    "summary": "백엔드 개발자, 스타트업 경력 3년"
  }
  ```
- 관련 테이블: `resumes`, `resume_text_contents`, `resume_file_contents`
- 벡터 DB: `user_document_chunks` → [vector-db-schema.md](../vector-db-schema.md)
