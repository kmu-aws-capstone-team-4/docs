# UC-USR-005 이메일 인증 링크 발송 화면 접근

| 항목 | 내용 |
|------|------|
| **액터** | 로그인 상태이나 이메일 미인증 사용자 |
| **목표** | 이메일 인증 링크 발송 화면에 접근하고, 필요 시 인증 링크를 재전송한다 |
| **사전 조건** | 계정이 생성되어 있으나 이메일 인증이 완료되지 않은 상태 |
| **사후 조건** | 인증 링크가 이메일로 전송되거나, 이미 전송된 링크 안내가 표시된다 |

---

## 기본 흐름

1. 사용자가 서비스 내 인증이 필요한 임의의 화면에 접근을 시도한다
2. 시스템이 해당 사용자의 이메일 인증 여부를 확인한다
3. 이메일 미인증 상태이면 요청한 화면 대신 이메일 인증 링크 발송 화면으로 강제 이동한다
4. 화면에 현재 계정의 이메일 주소와 함께 인증 링크 재전송 버튼을 표시한다
5. 사용자가 재전송 버튼을 클릭한다
6. 시스템이 인증 링크를 해당 이메일로 전송한다
7. 시스템이 전송 완료 안내 메시지를 표시한다

---

## 대안 흐름

- **5a.** 직전 전송으로부터 일정 시간이 경과하지 않은 경우, 재전송 버튼을 비활성화하고 남은 대기 시간을 표시한다
- **7a.** 사용자가 이메일의 인증 링크를 클릭하면 인증이 완료되고 프로필 작성 화면으로 이동한다 → [UC-USR-002-프로필_최초_작성.md](./UC-USR-002-프로필_최초_작성.md)

## 비고

- 이메일 인증이 완료되지 않은 사용자는 로그인 상태와 무관하게 인증이 필요한 모든 화면에서 이 화면으로 리다이렉트된다
- 인증이 필요하지 않은 화면(소개 페이지 등)은 리다이렉트 대상에서 제외된다 → [UC-NAV-001-소개_페이지_메인_화면_접근.md](./UC-NAV-001-소개_페이지_메인_화면_접근.md)
