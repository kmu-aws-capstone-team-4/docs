# 용어 정리 사전 (Glossary)

> 가상면접 웹 서비스에서 사용되는 주요 용어 및 개념 정의

---

## 사용자 및 계정

### 사용자 유형 (Actor Types)

| 한국어 | 영어명 | 설명 |
|--------|--------|------|
| **미가입 방문자** | `Guest` | 회원가입 또는 로그인 전 상태의 방문자 |
| **이메일 미인증 사용자** | `UnconfirmedUser` | 회원가입은 완료했으나 이메일 인증 링크를 클릭하지 않은 상태 (`is_email_verified = false`) |
| **프로필 미작성 사용자** | `OnboardingUser` | 이메일 인증은 완료했으나 프로필을 아직 작성하지 않은 상태 (`is_profile_complete = false`) |
| **인증 사용자** | `AuthenticatedUser` | 이메일 인증 및 프로필 작성까지 모두 완료한 사용자의 총칭. `FreeUser`와 `ProUser`를 포함 |
| **일반 사용자** | `FreeUser` | 인증 사용자 중 무료 요금제를 이용 중인 사용자 |
| **Pro 사용자** | `ProUser` | 인증 사용자 중 Pro 요금제를 구독 중인 사용자 |

### 계정 관련 용어

| 용어 | 영어 키워드 | 정의 |
|------|------------|------|
| **사용자 (User)** | `User` | 서비스에 가입하여 가상면접 기능을 이용하는 주체 |
| **회원가입 (Sign Up)** | `sign_up` | 이메일과 비밀번호를 입력하여 계정을 생성하는 절차 |
| **로그인 (Sign In)** | `sign_in` | 이메일과 비밀번호로 서비스에 인증하는 절차. `login`, `register` 키워드는 사용하지 않음 |
| **로그아웃 (Sign Out)** | `sign_out` | 현재 세션을 종료하는 절차 |
| **이메일 인증 (Email Confirmation)** | `confirm`, `confirmation`, `email_confirmation` | 회원가입 후 입력한 이메일 주소로 인증 링크를 전송하고, 사용자가 해당 링크를 클릭하여 이메일 소유를 확인하는 절차. `verify/verification` 대신 `confirm/confirmation`을 표준 키워드로 사용 |
| **인증 완료 (Confirmed)** | `is_confirmed`, `confirmed_at` | 이메일 인증 링크 클릭으로 인증이 완료된 상태. DB 컬럼명 등에서 `is_email_verified` 대신 `is_confirmed` 사용을 권장 |
| **인증 링크 (Confirmation Link)** | `confirmation_token`, `confirmation_link` | 이메일 인증을 위해 사용자의 이메일로 전송되는 일회성 URL |
| **온보딩 (Onboarding)** | `onboarding`, `is_onboarding_complete` | 이메일 인증 완료 후 프로필 작성까지의 초기 설정 단계. 완료 시 `AuthenticatedUser`로 전환됨 |
| **프로필 (Profile)** | `profile`, `is_profile_complete` | 사용자의 이름, 희망 직군, 직업 등 개인 정보를 담은 데이터 집합. 이메일 인증과 별개로 작성 절차가 진행됨 |
| **이름 (Name)** | `name` | 서비스 내에서 사용자를 식별하는 표시 이름(본명 작성을 권장하도록 함) |
| **직군 (Job Category)** | `job_category` | 사용자가 지원하고자 하는 직업 분야 (예: IT/개발, 마케팅, 금융, 의료 등). 전체 사회 직군을 포괄하며 면접 질문 생성에 반영됨 |
| **직업 (Occupation)** | `occupation` | 사용자의 현재 직업 또는 직책. 면접 질문 맥락 설정에 활용됨 |
| **개인정보 수정 (Profile Edit)** | `profile_edit`, `update_profile` | 비밀번호, 닉네임, 희망 직군, 직업 등 프로필 정보를 변경하는 기능 |

## 요금제

| 용어 | 정의 |
|------|------|
| **무료 요금제 (Free Plan)** | 기본 기능만 제공되는 무료 이용 플랜. 일부 고급 기능 사용에 제한이 있음 |
| **Pro 요금제 (Pro Plan)** | 유료 구독 플랜. 무료 요금제 대비 추가 기능 및 더 많은 사용 횟수를 제공 |
| **기능 제한 (Feature Restriction)** | 요금제에 따라 특정 기능의 사용 가능 여부 또는 횟수가 제한되는 정책 |

## 이력서 및 지원 정보

| 용어 | 정의 |
|------|------|
| **이력서 (Resume)** | 사용자가 업로드하는 경력, 학력, 기술 등을 담은 문서. 면접 질문 생성의 맥락 자료로 활용됨 |
| **채용공고 (Job Posting)** | 사용자가 지원하고자 하는 특정 기업의 채용 공고 정보. 직접 입력하거나 URL/파일로 업로드 가능 |
| **지원 희망 기업 (Target Company)** | 사용자가 면접을 준비하는 대상 기업 |
| **사용자 컨텍스트 (User Context)** | 이력서, 채용공고, 직군/직업 정보 등 사용자와 관련된 모든 맥락 정보의 총칭. 면접 질문 생성 및 피드백에 활용됨 |

## AI 및 데이터 처리

| 용어 | 정의 |
|------|------|
| **LLM (Large Language Model)** | 대규모 언어 모델. 면접 주제 생성, 질문 생성, 꼬리 질문 생성, 피드백 리포트 작성 등에 활용되는 AI 모델 |
| **벡터 DB (Vector Database)** | 텍스트나 문서를 벡터(수치 표현)로 변환하여 저장하고, 의미적 유사도 기반으로 검색할 수 있는 데이터베이스. 사용자 업로드 자료 및 시스템 기준 자료 검색에 활용됨 |
| **RAG (Retrieval-Augmented Generation)** | 벡터 DB에서 관련 자료를 검색(Retrieval)한 후 이를 LLM 생성(Generation)에 활용하는 방식. 질문의 정확성과 맥락 적합성을 높임 |
| **STT (Speech-to-Text)** | 음성을 텍스트로 변환하는 기술. 사용자의 면접 답변을 실시간으로 텍스트로 변환하며, 크롬 브라우저의 Web Speech API를 활용 |
| **TTS (Text-to-Speech)** | 텍스트를 음성으로 변환하는 기술. 가상 면접관이 질문을 소리로 전달할 때 서버에서 생성하여 제공 |
| **경향 분석 (Trend Analysis)** | 시스템이 사용자의 이력서, 채용공고, 면접 이력 등을 분석하여 사용자의 최근 관심 분야 및 준비 방향을 자연어 형식으로 누적·파악하는 기능 |

## 가상 면접 세션

| 용어 | 정의 |
|------|------|
| **가상 면접 (Virtual Interview)** | AI 기반의 면접관과 사용자가 비디오·음성으로 진행하는 모의 면접 세션 |
| **면접 세션 (Interview Session)** | 하나의 가상 면접이 시작되어 종료될 때까지의 단위 |
| **면접 주제 (Interview Topic)** | LLM이 사용자 정보를 기반으로 생성하는 면접 질문의 대주제 (예: 프로젝트 경험, 기술 역량, 지원 동기 등) |
| **면접 질문 (Interview Question)** | 면접 주제와 관련 자료를 바탕으로 LLM이 생성하는 실제 질문 |
| **꼬리 질문 (Follow-Up Question, FollowUpQuestion)** | 사용자의 답변 내용, 이전 질문, 사용자 관련 자료를 종합하여 생성되는 심화 질문. 1~3개 생성 가능 |
| **답변 (Answer)** | 사용자가 면접 질문에 대해 음성으로 제공하는 응답. STT를 통해 텍스트로 변환됨 |
| **가상 면접관 (Virtual Interviewer)** | 화면 중앙에 표시되는 SVG 기반의 인간형 캐릭터. TTS 음성과 함께 질문을 전달하며 실제 면접관처럼 응대함 |

---

## 면접 방식

| 용어 | 정의 |
|------|------|
| **꼬리질문 방식 (Follow-up Mode)** | 하나의 주제에 대해 초기 질문 후 사용자 답변을 기반으로 꼬리 질문을 1~3개 생성하며 심층 면접을 진행하는 방식 |
| **전체 프로세스 방식 (Full Process Mode)** | 자기소개, 지원동기, 기타 질문을 포함하여 실제 면접의 시작부터 끝까지 전 과정을 연습하는 방식. 꼬리 질문은 제공되지 않음 |
| **연습 모드 (Practice Mode)** | 사용자가 준비가 되었을 때 버튼을 눌러 답변을 시작하는 방식. 심리적 부담 없이 연습 가능 |
| **실전 모드 (Real Mode)** | 질문 제시 후 5~30초 사이의 랜덤 시간이 지나면 사용자의 버튼 클릭 여부와 무관하게 자동으로 답변 녹화가 시작되는 방식. 실제 면접 환경을 모사함 |
| **랜덤 대기 시간 (Random Wait Time)** | 실전 모드에서 질문 제시 후 답변 시작까지의 무작위 대기 시간. 5초~30초 범위 |

## 피드백 및 분석

| 용어 | 정의 |
|------|------|
| **면접 리뷰 리포트 (Interview Review Report)** | 가상 면접 세션 종료 후 제공되는 종합 피드백 문서. 답변 내용, 침묵 빈도, 시선 분산 등 다양한 지표를 포함 |
| **침묵 감지 (Silence Detection)** | 사용자가 답변 도중 특정 데시벨 이하로 5초 이상 정적이 유지될 경우 이를 감지하고 횟수 및 맥락을 기록하는 기능 |
| **시선 추적 (Gaze Tracking)** | 사용자의 고개 방향 또는 시선이 화면에서 벗어나는 빈도(분당 횟수)를 측정하는 기능. 피드백 리포트에 반영됨 |
| **데시벨 임계값 (Decibel Threshold)** | 침묵으로 판단하는 음량 기준값 |

## 스트릭 및 보상

| 용어 | 정의 |
|------|------|
| **스트릭 (Streak)** | 사용자가 매일 가상면접에 참여한 연속 일수를 기록하는 시스템. 꾸준한 학습을 독려하는 게이미피케이션 요소 |
| **스트릭 보상 (Streak Reward)** | 일정 스트릭 달성 시 지급되는 보상. Pro 요금제 전용 기능의 사용 횟수 등이 포함될 수 있음 |
| **게이미피케이션 (Gamification)** | 게임적 요소(스트릭, 보상 등)를 서비스에 도입하여 사용자의 지속적인 참여를 유도하는 전략 |

## 데이터 및 개인정보

| 용어 | 정의 |
|------|------|
| **영상 데이터 (Video Data)** | 가상 면접 중 수집되는 사용자의 영상 녹화 데이터 |
| **음성 데이터 (Audio Data)** | 가상 면접 중 수집되는 사용자의 음성 녹음 데이터 |

| **개인정보처리방침 (Privacy Policy)** | 서비스가 수집·이용·보관하는 개인정보의 종류, 목적, 기간 등을 명시한 문서 |
| **이용약관 (Terms of Service)** | 서비스 이용에 관한 권리·의무 및 규칙을 정의한 문서 |

## 시스템 사전 수집 데이터 (System Pre-collected Data)

> 서비스 운영 전 또는 운영 중 시스템이 사전에 구축·관리해야 하는 참조 데이터 목록.
> 사용자가 직접 입력하는 데이터가 아니라, 자동화 파이프라인 또는 어드민이 수집·등록하는 데이터이다.

### 기업 및 직무 정보

| 용어 | 내부 식별자 | 정의 |
|------|-----------|------|
| **기업 (Company)** | `companies` | 채용공고와 연결되는 기업 정보. 기업명, 산업군, 규모(대기업/중견/스타트업 등), 기업 소개 등을 포함. 사용자가 지원 희망 기업을 선택하거나 채용공고를 등록할 때 참조됨 |
| **산업군 (Industry)** | `industries` | 기업이 속한 산업 분류 (예: IT/소프트웨어, 금융/핀테크, 제조, 의료/바이오 등). 기업 및 직군과 연결되어 면접 질문의 산업 맥락을 구성함 |
| **직군 (Job Category)** | `job_categories` | 전체 사회 직업을 포괄하는 대분류 직군 목록 (예: IT/개발, 마케팅/광고, 금융/회계, 의료/간호, 교육, 영업, 디자인 등). 사용자 프로필의 희망 직군 선택지로 제공되며 면접 질문 생성의 핵심 필터로 활용됨 |
| **채용공고 (Job Description)** | `job_descriptions`, `user_job_descriptions` | 시스템이 사전 수집하거나 사용자가 URL로 등록한 채용공고 원문 정보. `job_descriptions`는 URL 기준 공유 원문(크롤링 1회)이며, `user_job_descriptions`는 사용자별 북마크 레이어다. 기업명, 직무 내용, 자격 요건, 우대 사항 등을 포함하며 면접 질문 생성의 맥락 자료로 활용됨. 사용자 프로필의 `Occupation`(현재 직업)과 혼동하지 않도록 주의 |
| **채용 유형 (Employment Type)** | `employment_types` | 고용 형태 분류 (예: 정규직, 계약직, 인턴, 프리랜서 등). 채용공고 및 면접 맥락 설정에 활용됨 |
| **경력 수준 (Career Level)** | `career_levels` | 경력 단계 분류 (예: 신입, 1~3년차, 3~5년차, 5년 이상, 임원급 등). 면접 질문 난이도 및 유형 결정에 활용됨 |
| **지원 현황 (Application Context)** | `user_application_contexts` | 사용자가 특정 기업·채용공고·면접 단계를 묶어 등록한 지원 준비 단위. 가상 면접 시작 시 선택지로 제공되며, 선택된 컨텍스트는 면접 주제 및 질문 생성에 반영됨 |
| **면접 단계 (Interview Stage)** | `interview_stages` | 채용 프로세스 내 면접의 진행 단계 (예: 1차 면접, 2차 면접, 임원 면접, 최종 면접, 인성 면접, 기술 면접 등). 단계에 따라 면접 질문의 유형과 깊이가 달라짐 |

### 면접 지식 베이스

| 용어 | 내부 식별자 | 정의 |
|------|-----------|------|
| **면접 질문 유형 (Question Category)** | `question_categories` | 면접 질문의 유형 분류. `technical`(기술), `behavioral`(행동/경험), `situational`(상황 판단), `motivation`(지원동기), `self_intro`(자기소개), `culture_fit`(조직 적합성) 등. 질문 생성 및 리포트 분류에 활용됨 |
| **면접 질문 난이도 (Question Difficulty)** | `question_difficulties` | 질문의 난이도 등급. `easy`, `medium`, `hard` 3단계로 구분. 사용자 경력 수준에 맞는 질문 생성 시 필터로 활용됨 |
| **핵심 역량 (Core Competency)** | `core_competencies` | 직군·직무별로 면접에서 평가하는 핵심 역량 항목 (예: 문제 해결력, 커뮤니케이션, 기술 전문성, 리더십 등). 면접 주제 생성 및 피드백 리포트의 평가 기준으로 활용됨 |
| **면접 질문 예시 (Interview QA Example)** | `interview_qa_examples` (벡터 DB) | 직군·직무별로 실제 면접에서 출제되는 질문과 모범 답변 예시 쌍. 벡터 DB에 저장되어 RAG 파이프라인에서 질문 생성 품질 향상에 활용됨 |
| **시스템 지식 청크 (System Knowledge Chunk)** | `system_knowledge_chunks` (벡터 DB) | 직군별 면접 트렌드, 핵심 역량 설명, 산업 동향, 면접 에티켓 등 시스템이 사전 구축한 지식 베이스. 벡터 DB에 저장되어 사용자 업로드 자료가 부족할 때 보완 맥락으로 제공됨 |

### 요금제 및 기능 정책

| 용어 | 내부 식별자 | 정의 |
|------|-----------|------|
| **기능 키 (Feature Key)** | `feature_quotas.feature_key` | 요금제별 사용 한도를 관리하는 기능의 식별자. 예: `follow_up_mode`(꼬리질문 방식), `full_process_mode`(전체 프로세스 방식), `gaze_tracking`(시선 추적), `real_mode`(실전 모드). 기능 정책 변경 시 코드 수정 없이 DB에서 관리됨 |
| **기능 한도 유형 (Quota Type)** | `feature_quotas.quota_type` | 기능 사용 한도의 집계 단위. `daily`(일별), `monthly`(월별), `unlimited`(무제한), `disabled`(사용 불가) 4가지로 구분됨 |
| **스트릭 보상 기준 (Streak Milestone)** | `streak_reward_logs.streak_milestone` | 보상이 지급되는 연속 참여 일수 기준값 (예: 7일, 14일, 30일). 운영팀이 사전에 정의하며, 달성 시 자동으로 보상이 지급됨 |
| **스트릭 보상 정책 (Streak Reward Policy)** | `streak_reward_policy` | 마일스톤 일수, 보상 유형, 대상 기능 키, 지급 횟수를 정의하는 운영 정책 테이블. 코드 배포 없이 운영팀이 동적으로 추가·수정·삭제할 수 있으며, 변경 즉시 다음 스트릭 적립부터 반영됨 |

| 용어 | 내부 식별자 | 정의 |
|------|-----------|------|
| **약관 버전 (Consent Version)** | `user_consents.consent_version` | 이용약관, 개인정보처리방침, AI 학습 데이터 활용 동의 등 각 약관의 버전 식별자 (예: `2025-01`). 약관 내용이 변경될 때 버전을 갱신하며, 사용자에게 재동의를 요청하는 기준이 됨 |
