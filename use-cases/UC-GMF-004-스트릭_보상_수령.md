# UC-GMF-004 스트릭 보상 수령

| 항목 | 내용 |
|------|------|
| **액터** | 시스템, 일반 사용자, Pro 사용자 |
| **목표** | 특정 스트릭 달성 시 보상을 자동 지급하고, 운영팀이 보상 정책을 동적으로 관리할 수 있도록 한다 |
| **사전 조건** | 스트릭 적립 완료 ([UC-GMF-001](./UC-GMF-001-스트릭_적립.md)) |
| **사후 조건** | 보상이 사용자 계정에 지급되고 지급 이력이 기록된다 |

---

## 기본 흐름

### 보상 자동 지급 (사용자 관점)

1. 스트릭 적립 후 시스템이 현재 `current_streak`을 보상 정책 목록과 비교한다
2. 달성된 마일스톤이 존재하고 해당 마일스톤에 대해 아직 보상이 지급되지 않은 경우 보상을 지급한다
3. 보상 유형에 따라 처리한다
   - `feature_usage_count`: 지정된 기능의 사용 가능 횟수를 `user_feature_usages`에 추가한다
4. 지급 이력을 `streak_reward_logs`에 기록한다
5. 사용자에게 보상 지급 알림을 표시한다

### 보상 정책 관리 (운영팀 관점)

1. 운영팀이 보상 정책 관리 화면에 접근한다
2. 마일스톤 일수, 보상 유형, 보상 대상 기능 키, 지급 횟수를 설정한다
3. 시스템이 정책을 저장한다
4. 이후 스트릭 적립 시 갱신된 정책이 즉시 반영된다

---

## 대안 흐름

- **2a.** 동일 마일스톤에 대해 이미 보상이 지급된 이력이 있는 경우 중복 지급하지 않는다
- **2b.** 하나의 스트릭 적립으로 여러 마일스톤을 동시에 달성한 경우 모두 지급한다

---

## 보상 정책 데이터 구조

보상 정책은 DB 테이블로 관리되어 코드 배포 없이 운영팀이 동적으로 조정할 수 있다.

| 필드 | 설명 | 예시 |
|------|------|------|
| `streak_milestone` | 보상 지급 기준 연속 일수 | `7`, `14`, `30`, `60` |
| `reward_type` | 보상 유형 | `feature_usage_count` |
| `reward_feature_key` | 보상 대상 기능 식별자 | `gaze_tracking`, `real_mode` |
| `reward_amount` | 지급 횟수 | `3`, `5`, `10` |

---

## 비고

- 보상 정책은 `streak_reward_policy` 테이블(또는 동등한 설정 테이블)에서 관리되며, 운영팀이 마일스톤·보상 유형·지급량을 자유롭게 추가·수정·삭제할 수 있다
- 지급 이력은 `streak_reward_logs`에 영구 보존되어 중복 지급 방지 및 감사 추적에 활용된다
- 관련 테이블: `streak_reward_logs`, `user_feature_usages`, `streak_reward_policy`
