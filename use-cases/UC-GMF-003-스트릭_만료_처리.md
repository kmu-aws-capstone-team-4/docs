# UC-GMF-003 스트릭 만료 처리

| 항목 | 내용 |
|------|------|
| **액터** | 시스템 |
| **목표** | 전날 가상 면접에 참여하지 않은 사용자의 스트릭을 초기화한다 |
| **사전 조건** | 매일 자정(00:00) 이후 스케줄러가 실행됨 |
| **사후 조건** | 미참여 사용자의 `current_streak`이 0으로 초기화된다 |

---

## 기본 흐름

1. 시스템 스케줄러가 매일 자정 이후 만료 처리 작업을 실행한다
2. 시스템이 `last_participated_date`가 전날 이전인 사용자를 조회한다
3. 해당 사용자들의 `current_streak`을 0으로 초기화한다
4. `longest_streak`은 변경하지 않는다 (역대 최장 기록은 보존)

---

## 비고

- `longest_streak`은 초기화 대상이 아니며 영구 보존된다
- 만료 처리는 사용자에게 푸시 알림 또는 다음 로그인 시 안내 메시지로 알릴 수 있다 (선택적 구현)
- 관련 테이블: `streaks`
